# AGENTS.md

## Project

名称：Chat Screeenshot

目标：开发一个 VS Code 插件，用于把 VSCode 中 Codex 聊天窗口里的单段回复或多段回复转换为截图，并支持手动复制 PNG 到系统剪贴板。

## MVP Decision

MVP 交互路径：
1. 用户在 Codex 聊天窗口中选中一段或多段回复并复制。
2. 执行扩展命令。
3. 扩展从系统剪贴板读取文本。
4. 扩展在 webview 中渲染聊天卡片并生成 PNG。
5. 用户手动点击 `复制 PNG` 将最新截图写入系统剪贴板。
6. 若手动复制失败，仍可使用下载兜底。

MVP 不做：
- 不依赖未公开的聊天内部数据结构或 DOM 注入。
- 不承诺直接“无复制步骤”抓取任意第三方聊天 participant 的历史消息。
- 不承诺在 Codex 聊天面板内直接勾选任意历史消息后由扩展读取，因为当前缺少稳定公开 API。

## Collector Workflow

新的目标交互：
1. 用户执行命令，打开常驻收集器面板。
2. 用户在 Codex 中选中需要的回复并复制。
3. 面板自动读取最新剪贴板文本，并把它加入收集列表。
4. 面板展示已收集内容与最终渲染预览。
5. 用户可以删除某一段、清空全部，或继续追加更多段。
6. 每次列表变化后，都会刷新最新 PNG 预览。
7. 用户手动点击 `复制 PNG`。
8. 用户把最新 PNG 粘贴到目标位置。

约束说明：
- VS Code 扩展没有标准的“独立悬浮窗”能力。
- 当前实现将使用最接近目标的常驻 `WebviewPanel` 作为收集器窗口。

## Current Implementation

已实现内容：
- 扩展命令 `codexChatScreenshot.openCollector`
- 默认快捷键打开收集器
- 常驻 `WebviewPanel` 收集器界面
- 自动监听系统剪贴板并收集新复制内容
- 多段回复列表、逐条删除、清空全部
- 基础 markdown 到卡片 HTML 的轻量渲染
- 直接 canvas 绘制 PNG 导出
- 手动复制 PNG
- 手动复制失败后的下载兜底
- 命令执行时直接读取系统剪贴板，不应要求用户再次手动粘贴

待修复问题：
- 仍需在真实 VS Code 环境中验证手动复制 PNG 是否足够稳定

图片复制的当前判断：
- VS Code 扩展宿主只公开文本剪贴板 API
- 图片复制目前走的是 webview 内的浏览器剪贴板接口
- 该接口通常受“安全上下文 + 用户手势 + 宿主实现”共同限制
- 因此当前产品决定仅保留“手动点击复制 PNG”这一条更稳定的路径

关键概念：
- `current participant` = 当前处理这次 chat request 的聊天参与方
- 它通常是某个具体助手或扩展注册的 chat participant
- 它不等于用户当前看到的整个聊天面板
- 因此，`ChatContext.history` 不能直接当成“读取当前 Codex 会话全部消息”的接口

渲染抓取方向的当前判断：
- 普通 VS Code 扩展没有公开 API 可以直接读取工作台里另一块 UI 的渲染 DOM
- 普通 VS Code 扩展也没有公开 API 可以直接对任意工作台区域执行元素级截图
- 如果走渲染抓取，更可能依赖系统级截图、辅助功能树或自动化工具，而不是标准扩展 API

当前 UI 方向：
- 收集器面板仅保留条目摘要、实时预览和关键操作按钮。
- 去掉不影响主流程的说明性文案、时间戳和冗余元信息。
- 界面与导出 PNG 都优先使用 VS Code 提供的主题变量，降低与用户主题冲突的概率。
- 第二轮视觉上采用“编辑式样张 / 校样板”方向，强化字体层次、版式张力和预览成品感，但不增加新的交互负担。
- 当前界面已使用更强的编辑性布局、响应式层次和主题适配细节，后续重点转回真实环境观感验证。
- 下一轮将把视觉方向从“带设计感的校样板”进一步收敛到“克制的 proofing utility”，减少炫技感并突出内容结构。
- 当前界面已转向更克制的 `proofing utility` 风格，重点从装饰性层叠转回内容和格式辨识度。

## Decision Log

- 决定先以 `AGENTS.md` 作为本项目的开发管理入口。

- 决定放弃 `foreignObject` 截图导出路径，改为直接 canvas 绘制 PNG。
- 决定继续以 clipboard-first 作为稳定交互路径，直到 VS Code/Codex 提供可依赖的聊天选择或读取接口。
- 决定将产品交互升级为常驻收集器面板，而非每次手动粘贴的单次预览面板。
- 决定提供默认快捷键打开收集器，贴近目标操作流。
- 决定移除自动复制 PNG，只保留手动复制 PNG。
- 决定将收集器 UI 收敛为极简布局，优先保证复制收集和预览效率，而不是展示辅助信息。
- 决定让导出 PNG 跟随 VS Code 当前主题变量，避免继续固定为深色导出风格。
- 决定在保持信息极简的前提下，引入更明确的视觉语言，而不是继续停留在普通工具面板风格。
- 决定采用 theme-adaptive 的“editorial utility”视觉方向，让预览面板更像校样板而不是常规设置页。
- 决定根据真实反馈收回部分装饰效果，把重点转移到内容可读性、特殊格式辨识度和摘要区紧凑度。
- 决定删除无信息量的摘要卡标题，让摘要列更像“索引”而不是“卡片列表”。
